Skill Reference

01 — INSTALLATION

Where it lives and how to install it

The Zendesk AI Slides skill is a Claude Code skill file: zendesk-slides-skill.md. Place it in your project's .claude/skills/ directory or in the global skills directory at ~/.claude/skills/.

Installation:

# Project-specific (recommended for team sharing) mkdir -p .claude/skills cp zendesk-slides-skill.md .claude/skills/ # Global (available across all projects) mkdir -p ~/.claude/skills cp zendesk-slides-skill.md ~/.claude/skills/

Once installed, the skill activates automatically when Claude detects slide, deck, or presentation work. No manual invocation required.

02 — WORKED EXAMPLE

Using the skill: starter prompt to draft output

Starter prompt:

I need a 10-slide deck for the Zendesk Intelligence Assistant launch. Internal audience (product + leadership). Sections: problem, solution, product, proof, close. Images: I have product screenshots for slides 6-7, customer photo for slide 8. Rest can be placeholders.

Claude's draft output (excerpt):

**Slide 1: Cover (speaker: yes)** Eyebrow: [Leave blank or "01 · Launch"] Headline: Zendesk Intelligence Assistant Speaker: Maya Chen, VP Product, Zendesk Image: [placeholder: product dashboard screenshot] --- **Slide 3: Section divider (body: yes)** Eyebrow: 01 · The problem Headline: The support gap Body: Today's agents spend 60% of their time searching for answers instead of helping customers. --- **Slide 6: Content (columns: 3, media: none)** Eyebrow: 03 · Product Headline: Built on three principles Subline: Context-aware, instant, and always learning Columns: - **Context-aware**: Understands full ticket history, not just keywords. - **Instant answers**: Sub-second response time for agent queries. - **Always learning**: Improves from every interaction automatically. --- Does this structure work, or should I adjust the archetype / split the content?

Common follow-ups:

"Make slide 6 a 4-column layout instead" → Claude adjusts parameters
"Add a Quote slide after slide 7" → Claude inserts and renumbers
"The headline on slide 3 is too vague" → Claude revises copy

03 — DRAFT BLOCK FORMAT

How Claude presents slide drafts

Each slide draft follows this structure:

**Slide [N]: [Archetype] ([parameters])** Eyebrow: [NN] · [SECTION LABEL] Headline: [headline text] Subline: [subline text, if applicable] Body: [body text or column content] Image: [image filename OR placeholder description] Speaker: [name, title, company — Cover only] Quote attribution: [name, title, company — Quote only]

The format is consistent and machine-parseable. Eyebrow numbering ([NN]) and section labels map to the deck's structure. Images reference the user's inventory or describe placeholders for later replacement.

04 — FULL SKILL CONTENT

The complete skill file

This is the skill Claude reads when drafting slides. Copy rules, archetype parameters, stop-and-ask discipline, and anti-patterns are all encoded here.

--- name: zendesk-slides description: Generate slide decks using the Zendesk AI Slides design system. Triggers on slide, deck, or presentation work. --- # Zendesk AI Slides Skill You draft slide content for the Zendesk AI Slides design system. Your job: pick the right archetype for each slide, write clear copy that fits the template constraints, and choose images from the inventory the user provides. The Figma template enforces all brand decisions — size, color, spacing, position. You enforce writing decisions — sentence case, no period on headlines, jargon swaps. You flag concerns about length and structure (slide is getting long, two ideas competing) for the user to resolve, but render what the user provides. ## Preflight Before drafting slides, confirm: 1. Theme applied: The user has applied the Zendesk AI Slides Figma template to their file 2. Image inventory: The user has identified available images or confirmed placeholder-only approach 3. Section structure: The user has outlined the deck's major sections 4. Confidentiality footer: Master element, do not modify 5. Audience context: Who will see this deck? Shapes jargon tolerance and detail density ## Archetype Set | Archetype | Parameters | When to use | |-----------|-----------|-------------| | Cover | speaker: yes/no | Deck opener | | Agenda | rows: 3–8 | Deck roadmap | | Section divider | body: yes/no | Chapter break | | Statement | — | Bold declarative slide | | Quote | — | Customer testimonial | | Content | columns: 1/2/3/4/5, media: image/icon/none | Flexible layout | | Product plate | — | Product feature with large image | | Data display | type: callouts | Metric callouts grid | | Timeline | milestones: 2–4 | Horizontal milestone sequence | | Closing | — | Final slide, Zendesk wordmark | ## Copy Rules 1. Sentence case for headlines 2. No period on headlines 3. One idea per slide (flag if two claims compete) 4. Body text gets periods 5. Brevity is the user's call (flag if dense) 6. Jargon swaps for external audiences ## Imagery Decision Rule 1. Reuse first (check inventory) 2. Placeholder second (describe what it should depict) 3. Never generate ## Stop-and-Ask List - Choosing a Closing slide vs. CTA slide - Splitting a dense slide - Using the Agenda archetype in short decks - Interpreting ambiguous source material ## Draft Block Format **Slide [N]: [Archetype] ([parameters])** Eyebrow: [NN] · [SECTION LABEL] Headline: [headline text] [etc.] ## Anti-Patterns 1. Empty columns (don't pad with filler) 2. Headline redundancy (eyebrow + headline repeat) 3. Mixing metaphors in Timeline (inconsistent granularity) 4. Burying the lede (conclusion in body, not headline) 5. Placeholder proliferation (pause after 3-4) 6. Agenda mismatch (agenda vs. section dividers)
